# Конвертация SDF/SMILES в PDBQT
Скрипты для подготовки множества лигандов к докингу (в AutoDock Vina). 
- `generate_pdbqt.py` - с использованием RDKit и OpenBabel.
- `meeko_generate_pdbqt.py` - с использованием RDKit и Meeko. Поддерживает гибкость циклов (>5-членных).

## Основные возможности
1. Конвертация нескольких молекул, записанных в один файл: SDF/SMILES в PDBQT.
2. Проверка на наличие дубликатов и их запись в разные файлы и папки.
3. 3D-генерация: Автоматическое добавление водородов, конвертация в 3D, минимизация энергии.
4. Гибкость: Алифатические циклы > 5 атомов помечаются как гибкие.

## Требования
Установка необходимых библиотек:

```
pip install numpy scipy rdkit vina meeko gemmi autogrid openbabel-wheel
```
## Использование
1. Поместите исходные файлы SDF/SMI и скрипт в рабочую папку.
2. Запустите нужный вам скрипт:

```
python meeko_generate_pdbqt.py
# или
python generate_pdbqt.py
```
3. Введите полный путь к файлам прямо в консоль.
4. Результаты появятся в папке converted_pdbqts/.

## Настройки (в коде)
В объекте **prepper** класса **MoleculePreparation** можно изменить параметры гибкости:
`min_ring_size=6`: минимальный размер кольца, которое будет считаться гибким

## Пример вывода программы
В архиве `converted_pdbqts.zip` находятся сконвертированные молекулы из обоих входных файлов

# Molecular Dynamics Preparation Script
`gromacs_pipeline.py` - скрипт для автоматизации подготовки малых молекул из SDF в форматы, совместимые с **GROMACS** (`.gro`, `topol.top`). Подготовка включает генерацию топологии с использованием силовых полей Open Force Field (openff-2.1.0.offxml).

## Основные возможности
1. Конвертация нескольких молекул, записанных в один файл: `.sdf` в `.gro` и `topol.top`.
2. Проверка на наличие дубликатов и их запись в разные файлы и папки.
3. 3D-генерация: Автоматическое добавление водородов, конвертация в 3D, минимизация энергии.
4. Валидация полученных файлов.

## Требования
Для работы скрипта требуется активировать окружение и установить библиотеки:

```
conda activate openff_env

conda install -c conda-forge \
    rdkit \
    openff-toolkit \
    openff-interchange \
    openff-forcefields \
    ambertools \
    openmm \
    gromacs
```

## Использование
1. Запускайте команду, находясь в директории с входными файлами:

```
python gromacs_pipeline.py
```
2. Введите полный путь к файлам прямо в консоль.

## Валидация файлов (grompp)
Чтобы проверить корректность созданных файлов и рассчитать начальную энергию системы, раскомментируйте последнюю строку скрипта:

```
validate_with_grompp(name)
```
При успешном выполнении появится следующий вывод:

```
"Validation passed for ваша-молекула"
```

## Параметры
* **Масштабирование:** Координаты автоматически конвертируются из Ангстрем (SDF) в Нанометры (GRO) с коэффициентом $0.1$.
* **Заряды:** Частичные заряды рассчитываются согласно выбранной модели в OpenFF (по умолчанию обычно AM1-BCC). Суммарный заряд молекулы проверяется на нейтральность.
* **Силовое поле:** Используются параметры OpenFF (параметризация `[ atomtypes ]` включена непосредственно в `topol.top`).


## Выходные файлы
* `CHEMBL45305.gro` — файл структуры GROMACS (координаты в нм).
* `topol.top` — топология системы (типы атомов, связи, углы, частичные заряды).
* `CHEMBL45305_pointenergy.mdp` — параметры для расчета энергии в одной точке (Single Point Energy), используются для валидации.
